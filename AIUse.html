Use of Artificial Intelligence in Blog Application Development

1. Overview

This document describes how Artificial Intelligence (AI) tools were used during the development of a blog application, with a particular focus on the admin functionality, frontend rendering, and AJAX-based interactions.

AI assistance was used as a development support tool to accelerate implementation, improve code quality, and validate architectural decisions. All final code was reviewed, adapted, and integrated manually by the developer.

2. Admin Functionality Development

AI was used to help design and implement the admin panel of the blog application. The admin panel allows administrators to:

  • Create, edit, and delete blog posts
  • Manage users and roles
  • View and manage subscriptions
  • Access all admin features through protected routes

AI assistance helped generate:

  • Express.js admin routes
  • Middleware logic for admin authorization
  • EJS templates for admin pages
  • Clean separation between API endpoints and rendered pages

3. Frontend Development (EJS + HTML + CSS)

AI was also used during frontend development to:

  • Structure EJS templates for admin and blog pages
  • Create simple, readable HTML layouts
  • Design a minimal and consistent CSS style for the admin interface

The frontend remains server-rendered using EJS, while JavaScript is used only where dynamic interaction is required (AJAX actions).

4. Use of AI for AJAX Functionality

A major area where AI assistance was used is the implementation of AJAX-based functionality. This allows the admin panel to perform actions without full page reloads.

4.1 AJAX Create / Edit / Delete for Posts

AI was used to help design JavaScript functions that:

  • Submit forms using fetch()
  • Send JSON payloads to Express endpoints
  • Handle POST, PUT, and DELETE HTTP methods
  • Update the UI after successful operations

4.2 AJAX Pagination of Posts

AI assistance was also used to design AJAX-based pagination for blog posts. This allows users to load additional posts dynamically without refreshing the page.

The AI helped with:

  • Pagination query parameters (page, limit)
  • Backend endpoint design for paginated results
  • Frontend JavaScript logic to fetch and append posts
  • Improving user experience through incremental loading

6. Role of AI in the Development Process

AI was used as a supporting development tool, not as a replacement for developer decision-making. The AI helped:

  • Speed up boilerplate generation
  • Provide architectural suggestions
  • Debug authentication and middleware issues
  • Improve clarity and structure of code

All AI-generated content was reviewed, tested, and modified as needed to meet project requirements and coding standards.

7. Conclusion

The use of AI significantly improved development efficiency and helped implement robust admin and frontend functionality for the blog application. By combining AI-generated guidance with hands-on development and debugging, the project achieved a clean, maintainable, and feature-complete result.
